# 反思总结：为什么一开始不使用 Jina Reader

**反思时间**: 2026-03-06 14:39
**问题**: 为什么不一开始使用 agent-reach 的 Jina Reader？

---

## ❌ 我的错误

### 错误 1: 工具发现失败
**问题**: 我没有检查"还有什么其他工具可以用"
**应该**: 先运行 `agent-reach doctor` 查看可用工具

### 错误 2: 优先级判断错误
**问题**: 优先使用 curl（原始工具），而非 Jina Reader（智能工具）
**应该**: 优先使用智能工具

### 错误 3: 违反第一性原理
**问题**: 没有问"什么是最优解决方案？"
**应该**: 
- 剥离表象: "curl 失败" → "需要能获取动态内容的方法"
- 最基础的事实: Jina Reader 可以执行 JavaScript
- 最优解: 使用 Jina Reader

---

## ✅ 正确的方法

### 步骤 1: 检查可用工具
```bash
agent-reach doctor
```

### 步骤 2: 选择最优工具
根据 `doctor` 输出选择工具：
- ✅ 任意网页 → Jina Reader
- ✅ 静态 HTML → curl
- ✅ 简单页面 → web_fetch

### 步骤 3: 执行访问
```bash
curl -s "https://r.jina.ai/<URL>"
```

---

## 🎯 关键洞察

### 1. 智能工具 > 原始工具 ⭐⭐⭐⭐⭐

**原始工具**:
- curl: 只能获取静态 HTML
- 不执行 JavaScript

**智能工具**:
- Jina Reader: 执行 JavaScript，返回 Markdown
- 可以访问单页应用

### 2. 检查 > 假设 ⭐⭐⭐⭐⭐

**错误**:
- 假设 "curl 是最好的方法"
- 没有检查其他工具

**正确**:
- 先检查 `agent-reach doctor`
- 根据实际情况选择工具

### 3. 第一性原理思考 ⭐⭐⭐⭐⭐

**问题**: "获取网页内容"
**本质**: "需要能获取动态内容的方法"
**最基础事实**: Jina Reader 可以执行 JavaScript
**最优解**: 使用 Jina Reader
