# Agent 矩阵使用指南

## 快速开始

### 方式1: 使用 orchestrate 工具（推荐）

直接告诉 Main Agent 你要做什么：

```
生成小红书内容，推荐5个AI工具
```

Main Agent 会自动调用 `orchestrate` 工具，创建 5 个 Sub-Agents 协同工作。

### 方式2: 指定具体流程

```
使用 Agent 矩阵帮我生成内容：
1. 分析需求：写一篇关于AI工具的文章
2. 收集2026年最新资料
3. 生成内容策略
4. 设计视觉风格
5. 质量审核
```

## 工作流程

### 标准流程（5个 Agents）

```
用户需求
  ↓
Agent 1: 需求理解 (60秒)
  - 输入: 自然语言需求
  - 输出: 结构化任务规范
  ↓
Agent 2: 资料收集 (120秒)
  - 输入: 任务规范 + 关键词
  - 输出: 12条2026年最新资料
  ↓
Agent 3: 内容策划 (90秒)
  - 输入: 任务规范 + 资料
  - 输出: 内容策略 + 执行计划
  ↓
Agent 4: 视觉生成 (60秒)
  - 输入: 内容策略 + 资料
  - 输出: 视觉参数
  ↓
Agent 5: 质量审核 (30秒)
  - 输入: 所有内容
  - 输出: 质量报告
  ↓
整合结果 → 返回用户
```

**总时间**: 约 4-7 分钟

### 快速流程（3个 Agents）

```
用户需求
  ↓
Agent 1: 需求理解 (60秒)
  ↓
Agent 2: 资料收集 (120秒)
  ↓
Agent 3: 内容生成 (90秒)
  ↓
返回内容（不含视觉和审核）
```

**总时间**: 约 3-5 分钟

## 调整和优化

### 调整超时时间

```
使用 Agent 矩阵，超时时间设置为：
- 需求理解: 90秒（默认60秒）
- 资料收集: 180秒（默认120秒）
```

### 指定平台

```
生成小红书内容，要求：
- 风格: 专业
- 长度: 200-300字
- emoji: 适量
- 标签: #AI工具 #效率提升
```

### 指定资料来源

```
收集资料时，优先使用：
- 网络搜索: metaso-search
- 技术文档: 官方文档
- 团队知识库: 飞书文档
```

### 并行执行

```
使用 Agent 矩阵并行执行：
1. 同时生成图片和视频
2. 同时生成小红书和抖音内容
```

## 常见问题

### Q: 需要多长时间？

**A**:
- 标准流程: 4-7分钟（5个 Agents）
- 快速流程: 3-5分钟（3个 Agents）
- 仅收集资料: 2-3分钟（2个 Agents）

### Q: 可以并行执行吗？

**A**: 可以！例如：
```
同时生成小红书和抖音版本
```

### Q: 如何查看 Sub-Agent 的详细过程？

**A**: 使用 `sessions_list` 和 `sessions_history`：
```
查看所有 Sub-Agent sessions
查看 requirement-agent 的历史
```

### Q: 失败了怎么办？

**A**:
- 每个 Agent 有独立超时
- 失败的 Agent 会显示错误信息
- 可以单独重试失败的 Agent

### Q: 可以自定义 Agent 吗？

**A**: 可以！创建新的 Skill，然后在 `sessions_spawn` 中指定 agentId。
