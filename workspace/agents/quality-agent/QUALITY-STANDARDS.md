# quality-agent - 质量审核标准配置

## 📋 各阶段质量标准

### 1. requirement_review（需求审核）

**质量阈值**: 80/100

**审核维度**:
- **需求完整性** (35%)
  - 平台信息 ✅
  - 主题描述 ✅
  - 风格要求 ⚠️ 可选

- **需求清晰度** (35%)
  - 需求明确 ✅
  - 无歧义 ✅
  - 可执行 ✅

- **需求可实现性** (30%)
  - 技术可行 ✅
  - 资源充足 ✅
  - 时间合理 ⚠️ 可选

**重试次数**: 2

**重试条件**: 得分 < 80

## 🎯 整体质量评分

### 计算公式

```
整体评分 = (需求质量 × 15%) + (资料质量 × 20%) + (文案质量 × 25%) + (图片质量 × 20%) + (视频质量 × 20%)
```

### 评级标准

| 整体评分 | 评级 | 说明 |
|---------|------|------|
| ≥ 90 | 优秀 | 质量很高，可以直接使用 |
| 85-89 | 良好 | 质量较好，可以微调使用 |
| 80-84 | 及格 | 质量一般，建议改进后使用 |
| < 80 | 不及格 | 质量较差，需要重新生成 |

## 📊 使用示例

### 调用 quality-agent

```bash
# 需求审核（使用默认标准）
bash /home/node/.openclaw/agents/quality-agent/scripts/review-v2.sh "requirement" "/path/to/requirement.md"

# 文案审核（使用自定义标准）
bash /home/node/.openclaw/agents/quality-agent/scripts/review-v2.sh "content" "/path/to/content.md" '{"quality": 90, "platform": "抖音"}'

# 图片审核（使用自定义标准）
bash /home/node/.openclaw/agents/quality-agent/scripts/review-v2.sh "visual" "/path/to/visual.md" '{"quality": 90, "style_match": true}'
```
