**时间**: 2026-03-12 16:00

---

## 🔍 问题现象

### 测试结果对比

| 测试 | 时间 | 结果 | Token 消耗 | 说明 |
|------|------|------|-----------|------|
| 测试 1 (15:39) | 执行 exec | ✅ 成功 | 35.7k | 简单任务 |
| 测试 2 (15:58) | 没有执行 exec | ❌ 超时 | 0 | 复杂任务 |

**任务差异**：
- 测试 1：简单 exec 测试
- 测试 2：生成图片任务（更复杂）

### 问题

**LLM 的行为不一致**：
- 简单任务：✅ 执行 exec
- 复杂任务：❌ 不执行 exec，只返回文字

---

## 💡 可能的原因

### 1. 任务复杂度 ⭐⭐⭐

**分析**：
- 简单任务（"echo..."）：LLM 能理解并执行
- 复杂任务（"生成图片"）：LLM 可能理解为"文字描述"

**解决**：
- 在 AGENTS.md 中提供完整的工作流程
- 包含具体的代码示例

### 2. LLM 理解偏差 ⭐⭐⭐⭐⭐

**分析**：
- LLM 可能认为"生成图片" = "用文字描述图片"
- LLM 没有意识到需要调用 API

**解决**：
- 更明确的指令："必须调用 exec"
- 提供完整的工作流程代码

### 3. 提示词不够明确 ⭐⭐⭐⭐

**分析**：
- 之前的指令太抽象
- 没有提供具体步骤
- LLM 不知道如何执行

**解决**：
- 提供完整的 bash 代码
- 包含 curl 命令示例
- 明确每一步操作

### 4. AGENTS.md 可能没有被正确读取 ⭐⭐⭐

**分析**：
- 子 Agent 使用 `promptMode=minimal`
- AGENTS.md 可能被截断
- 关键指令可能丢失

**解决**：
- 精简 AGENTS.md
- 核心指令放在前面
- 删除冗余内容

---

## 🎯 解决方案

### 方案 1：优化 AGENTS.md ⭐⭐⭐⭐⭐

**方法**：
- 精简内容
- 提供完整代码
- 强调"必须执行"

**实施**：
- ✅ 已更新为 v10.0（强制执行版）
- 提供完整的 bash 工作流程
- 明确禁止只返回文字

### 方案 2：创建脚本版本 ⭐⭐⭐⭐

**方法**：
- 创建 `agents/visual-agent/run.sh`
- Main Agent 直接调用脚本
- 不依赖 LLM Agent

**优势**：
- 100% 可靠
- Token 消耗为 0
- 执行时间可预测

**劣势**：
- 失去 LLM 的灵活性
- 需要维护脚本

### 方案 3：混合方案 ⭐⭐⭐⭐⭐

**方法**：
- Main Agent 判断任务类型
- 简单任务 → LLM Agent
- 复杂任务 → 脚本

**优势**：
- 结合两者优点
- 灵活选择
