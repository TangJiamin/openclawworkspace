## 🤔 失败原因分析

### 原因 1: 思考循环 ⭐⭐⭐⭐⭐

**现象**：
- Token 消耗为 0
- 返回"脚本正在生成..."
- 超时时间正好是限制（3 分钟）

**分析**：
- LLM 看到任务后开始"思考"如何完成
- 思考过程可能陷入循环
- 思考过程不计入 Token（思考不输出）
- 超时后返回当前的想法

**为什么简单测试成功**：
- 简单任务（echo）不需要思考
- LLM 直接执行
- Token 消耗正常（35.7k）

### 原因 2: 工具调用被阻止 ⭐⭐⭐⭐

**可能性**：
1. **工具列表未正确传递**
   - 虽然 `tools.subagents.tools` 已配置
   - 但可能没有传递到 LLM 的系统提示词
   - LLM 可能看不到 exec 工具

2. **LLM 不知道可以调用工具**
   - 系统提示词可能没有明确告诉 LLM
   - LLM 可能认为它没有权限

3. **安全策略阻止**
   - 可能有其他安全检查
   - 阻止了 LLM 调用工具

### 原因 3: LLM 理解偏差 ⭐⭐⭐

**可能性**：
1. **任务理解错误**
   - LLM 把"生成图片"理解为"描述图片"
   - 而不是"调用 API 生成图片"

2. **指令理解错误**
   - LLM 看到"调用脚本"不理解
   - LLM 可能在思考"为什么我要调用这个"

### 方案 2: 测试更简单的任务

**问题**：什么类型的任务会成功？

**方法**：
- 测试比 echo 更复杂的任务
- 测试比图片生成更简单的任务
- 找到成功/失败的边界

**实施**：
```bash

sessions_spawn(
    agent_id="visual-agent",
    task="读取 /home/node/.openclaw/workspace/agents/visual-agent/AGENTS.md 并总结前 3 行"

```

### 方案 3: 使用 ACP Agent

**问题**：编程 Agent 是否更可靠？

**方法**：
- 使用 `runtime="acp"` 而不是 `runtime="subagent"`
- ACP Agent 可以直接执行代码
- 可能更可靠

**实施**：
```python
sessions_spawn(
    agent_id="visual-agent",
    runtime="acp",
    task="生成图片..."
)
```
