## 🎯 重新规划的 Agent 矩阵

### 核心原则

**基于已有能力，避免重复建设**

- 优先复用已有 Skills
- 只创建缺失的关键能力
- 聚焦"编排"而非"实现"

### 简化架构（3层 × 6 Agent）

```
┌─────────────────────────────────────────────┐
│  输入层 (2 Agents)                           │
├─────────────────────────────────────────────┤
│  • 资料收集 Agent                            │
│  • 需求理解 Agent                            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  生产层 (3 Agents)                           │
├─────────────────────────────────────────────┤
│  • 内容策划 Agent                            │

│  • 视频生成 Agent                            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐

│  • 质量审核 Agent                            │
└─────────────────────────────────────────────┘
```

## 📋 Agent 详细设计

### 输入层

#### Agent 1: 资料收集 Agent (Material Collector)

**职责**: 收集和整理素材

**复用技能**:
- ✅ `ai-daily-digest` - 技术资讯收集
- ✅ `metaso-search` - 网络搜索
- ✅ `feishu-doc` - 飞书文档读取

**新增能力**: 无（已有技能足够）

**工作流程**:
```
用户需求 → 技术资讯抓取(ai-daily-digest)
        → 网络搜索补充(metaso-search)
        → 飞书文档读取(feishu-doc)
        → 整理成结构化资料
```

---

#### Agent 2: 需求理解 Agent (Requirement Analyzer)

**职责**: 将自然语言需求转化为结构化任务

**新增能力**: **需要创建**

**工作流程**:
```
自然语言需求 → 意图识别
            → 属性提取
            → 缺失信息追问
            → 生成任务规范(JSON)
```

**输出格式**:
```json
{
  "task_id": "uuid",
  "content_type": ["文案", "图片", "视频"],
  "platforms": ["小红书", "抖音"],
  "style": "专业/轻松/幽默",
  "tone": "友好/权威",
  "length": "100-200字",
  "quality_threshold": 85
}
```

### 生产层

#### Agent 3: 内容策划 Agent (Content Planner)

**职责**: 基于资料和需求生成内容策略

**复用技能**:
- ✅ `metaso-search` - 补充信息
- ✅ `ai-daily-digest` - 趋势参考

**新增能力**: **需要创建**

**工作流程**:
```
需求规范 + 资料 → 内容策略分析
              → 平台适配(小红书/抖音/微信)
              → 生成执行计划
              → 输出内容大纲
```

**输出格式**:
```json
{
  "content_strategy": {
    "hook": "用痛点开场",

    "keywords": ["AI工具", "效率提升"],
    "cta": "关注获取更多"
  },
  "production_plan": [
    {"agent": "文案生成", "input": {...}, "order": 1},
    {"agent": "视觉生成", "input": {"text": "文案"}, "order": 2},
    {"agent": "质量审核", "input": {"文案": "...", "图片": "..."}, "order": 3}
  ]
}
```

#### Agent 4: 视觉生成 Agent (Visual Generator)

**职责**: 生成图片、信息图、封面

**复用技能**:
- ✅ `visual-generator` - 多维参数系统（核心）
- ✅ `Seedance API` - 图片生成

**新增能力**: 无（已有技能足够）

**工作流程**:
```
内容需求 → visual-generator分析
         → 推荐风格和布局
         → 调用 Seedance API
         → 返回生成图片
```

**关键优势**:
- 自动内容分析
- 智能参数推荐
- 平台预设库

---

#### Agent 5: 视频生成 Agent (Video Generator)

**职责**: **生成分镜提示词和视频（必须先有图片）**

**⚠️ 关键原则**: 
- **必须先完成 visual-agent 生成图片**
- **根据图片 + 视频描述生成视频**
- **禁止并行生成图片和视频**
- **禁止直接从文本生成视频**

**复用技能**:
- ✅ `seedance-storyboard` - 分镜提示词生成（核心）
- ✅ `Seedance API` - 视频生成

**新增能力**: 无（已有技能足够）

**工作流程**:
```
视频需求 → 检查是否已有图片
         ↓ (如果没有)
         → 调用 visual-agent 生成图片 ⚠️ 必须完成
         ↓ (有图片后)
         → seedance-storyboard 引导
         → 完善细节(5个维度)
         → 生成分镜提示词
         → 调用 Seedance API (传入图片)
         → 返回生成视频
```

**关键优势**:
- 对话式引导
- 时间轴构建
- 多模态输入支持
- 专业镜头语言
- **保证视频基于高质量图片生成**

### 优化层

#### Agent 6: 质量审核 Agent (Quality Reviewer)

**职责**: **多维度质量检查（文案、图片、视频）**

**复用技能**:
- ✅ `OpenAI Moderation API` - 内容合规

**新增能力**: **需要创建**

**检查维度**:
- ✅ **文案质量**（连贯性、准确性、吸引力）
- ✅ **图片质量**（清晰度、风格匹配、品牌一致性）
- ✅ **视频质量**（流畅度、节奏感、镜头语言）
- ✅ **整体一致性**（文案-图片-视频的协调性）
- ✅ 平台合规（敏感词、版权、格式）
- ✅ 用户要求（长度、风格、目标）

**工作流程**:
```
待审核内容（文案 + 图片 + 视频）
  ↓
文案质量检查 → 图片质量检查 → 视频质量检查
  ↓              ↓                ↓
  ←——— 整体一致性检查 ——————→
  ↓
平台合规检查(OpenAI Moderation)
  ↓
用户要求匹配检查
  ↓
生成质量报告
  ↓
(不通过) 返回修改建议
```

**输出格式**:
```json
{
  "overall_score": 87,
  "passed": true,
  "issues": [],
  "suggestions": [],
  "checks": {
    "copywriting": {
      "quality": 90,
      "engagement": 85,
      "clarity": 92
    },
    "image": {
      "quality": 88,
      "style_match": 90,
      "brand_consistency": 85
    },
    "video": {
      "quality": 85,
      "flow": 88,
      "visual_language": 82
    },
    "overall_consistency": 87,
    "platform_compliance": 95,
    "requirement_match": 90
  }
}
```

## 🔄 编排模式

### 1. 线性流水线（最常用）

```
需求理解 → 内容策划 → 视觉生成 → 质量审核 → 输出
```

**适用**: 标准内容生产

### 2. 并行分支

```
内容策划 → ┌→ 文案生成 → 质量审核 ─┐
          ├→ 视觉生成 → 质量审核 ─┤ → 组合
          └                      ┘
          
⚠️ 注意：视频生成不能并行！
视频生成必须等待视觉生成完成：
视觉生成 → 图片 → 视频生成 → 视频 → 质量审核
```

**适用**: 同一素材多平台适配（但视频必须等图片）

### 3. 智能路由

```
需求理解 → (判断类型) → 路由到对应流程
                        ├→ 仅文案 → metaso搜索 → GLM生成
                        ├→ 仅图片 → visual-generator
                        └→ 视频 → seedance-storyboard
```

**适用**: 多样化需求
