## 🔍 根本原因分析

### 1. 超时设置不合理 ⭐⭐⭐⭐⭐

**video-agent 实际工作流**：
```
1. 生成 5-10 张分镜图片
   - 每张图片生成：30-60秒
   - 总耗时：150-600秒

2. 视频合成
   - 剪辑、拼接、转场：60-120秒

3. 后期处理
   - 字幕、BGM、特效：30-60秒

总计：240-780秒（平均 360秒）
```

**问题**：超时设置 120秒，仅为实际耗时的 33%

**research-agent 实际工作流**：
```
1. 多源搜索（Metaso + Brave）
   - 搜索：30秒
   - 结果整理：30秒

2. 信息验证
   - 交叉验证：30秒

3. 角度分析
   - 深度分析：30秒

4. 生成输出
   - 结构化输出：30秒

总计：120秒
```

**问题**：超时设置 120秒，刚好卡在边界，无安全余量

### 2. 串行执行效率低

**当前流程**（完全串行）：
```
requirement-agent (22秒)
  ↓
research-agent (117秒)
  ↓
content-agent (39秒)
  ↓
quality-agent (29秒)
  ↓
visual-agent (55秒)
  ↓
video-agent (119秒)

总耗时：381秒
```

**问题**：所有任务串行，无法利用并行能力

### 优化 2：并行执行 ⭐⭐⭐⭐

**并行组合分析**：

#### 组合 1：research-agent + requirement-agent（深度）✅

**依赖分析**：
- requirement-agent 需要先输出**初步需求**（10秒）
- research-agent 可以基于**初步需求**开始搜索
- requirement-agent 继续**深度分析**（12秒）
- 如果深度分析有重大调整，research-agent 补充搜索

**实施代码**（伪代码）：
```javascript
// Step 1: requirement-agent 快速分析
const quickReq = await requirementAgent.run({
  mode: 'quick',
  timeout: 10
})

// Step 2: 并行执行
const [deepReq, research] = await Promise.all([
  requirementAgent.run({
    mode: 'deep',
    timeout: 12,
    basedOn: quickReq
  }),
  researchAgent.run({
    basedOn: quickReq,
    timeout: 180
  })
])

// Step 3: 检查是否需要补充搜索
if (deepReq.hasMajorChange) {
  await research.supplement()
}
```

**效果**：
```
优化前：10秒 + 120秒 = 130秒
优化后：max(12秒, 120秒) = 120秒（无重大调整）
      12秒 + 30秒（补充搜索） = 42秒（有重大调整）
速度提升：7.7% - 67.7%
```

**风险评估**：
- ⚠️ 可能需要补充搜索（概率 < 10%）
- ✅ 即使补充搜索，总体仍更快

---

#### 组合 2：quality-agent（文案）+ visual-agent ✅

**依赖分析**：
- quality-agent 审核文案（30秒）
- visual-agent 生成分镜（180秒）
- **关键**：如果文案不达标，立即停止 visual-agent

**实施代码**（伪代码）：
```javascript
// 并行执行
const [quality, visual] = await Promise.all([
  qualityAgent.run({
    type: '文案审核',
    timeout: 30
  }),
  visualAgent.run({
    type: '生成分镜',
    timeout: 180,
    stoppable: true  // 可停止
  })
])

// 如果文案不达标，停止 visual-agent
if (quality.score < 80) {
  await visual.stop()
  // 重新生成文案
  await contentAgent.regenerate()
}
```

**效果**：
```
优化前：30秒 + 180秒 = 210秒
优化后：max(30秒, 180秒) = 180秒（文案达标）
      30秒（文案不达标，节省 150秒）
速度提升：14.3%（文案达标）
速度提升：85.7%（文案不达标）
```

**风险评估**：
- ⚠️ 文案不达标概率 ~20%
- ✅ 即使不达标，也节省了大量时间
