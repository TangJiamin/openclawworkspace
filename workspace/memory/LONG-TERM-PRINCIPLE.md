# 长期主义原则

## 🎯 核心原则

**长期主义优先于短期快速**

### 决策标准

**优先级**:
1. ✅ **长期可维护性** - 代码/系统可以长期使用和扩展
2. ✅ **高复用率** - 充分利用现有组件，避免重复造轮子
3. ✅ **质量保证** - 使用专业工具，确保输出质量
4. ⏰ **短期速度** - 在满足前三项的前提下考虑速度

### 禁止的行为

❌ **短期快速但低复用率的设计**:
- 绕过现有 Agents 直接调用底层 API
- 不使用专业 Skills 手动实现
- 重复造轮子而不是复用

❌ **"先快速实现，以后再优化"**:
- 这种思路会导致技术债
- "以后"通常永远不会来
- 系统会越来越难以维护

### 正确的行为

✅ **长期主义设计**:
1. **复用优先** - 优先使用现有的 Agents 和 Skills
2. **完整流程** - 遵循 Agent 矩阵的完整工作流
3. **质量第一** - 使用专业工具，确保输出质量
4. **可扩展性** - 设计要考虑未来扩展

### 具体要求

**必须**:
- 使用 requirement-agent 分析需求
- 使用 research-agent 收集资料
- 使用 visual-agent 生成视觉参数
- 使用 video-agent 生成分镜脚本
- 使用 quality-agent 进行质量审核

**禁止**:
- 绕过 Agent 层直接调用底层 API
- 跳过专业 Skills 手动实现
- 为了速度牺牲复用性和质量
