### 问题描述

**严重性**: ⭐⭐⭐⭐⭐ 最高优先级

**问题**:
1. ❌ 我在 2026-03-06 硬编码了 API Key 到脚本
2. ❌ 硬编码的值: `sk-53722ca0e9814f1afd24ac334d08e2f3c9b8ee2d1e4ac80f`
3. ❌ 这个值被记录到会话历史 `.jsonl` 文件

### 根本原因

**违反原则**: **禁止硬编码 API Key**

**错误代码**:
```bash

# ❌ 错误：直接导出
export XSKILL_API_KEY="sk-53722ca0e9814f1afd24ac334d08e2f3c9b8ee2d1e4ac80f"
```

### 防范措施

**核心原则**:
1. ⭐⭐⭐⭐⭐ **.env 文件是所有 API Key 的唯一来源**
2. ⭐⭐⭐⭐⭐ **绝对禁止硬编码 API Key**
3. ⭐⭐⭐⭐⭐ **绝对禁止设置默认值**

**正确做法**:
```bash
# ✅ 正确：从 .env 文件读取
set -a
source /home/node/.openclaw/.env 2>/dev/null || true

API_KEY="${XSKILL_API_KEY}"

if [ -z "$API_KEY" ]; then
    echo "❌ 错误: 未设置 XSKILL_API_KEY"
    echo "请在 .env 文件中设置: XSKILL_API_KEY=sk-your-api-key"
    exit 1
fi
```

**错误做法**:
```bash
# ❌ 错误：硬编码
API_KEY="sk-xxx"

# ❌ 错误：设置默认值
API_KEY="${XSKILL_API_KEY:-sk-xxx}"

# ❌ 错误：导出环境变量
export XSKILL_API_KEY="sk-xxx"
```

### 学习记录

**关键洞察**:
1. ⭐ **永远不要为了"方便"而设置默认值**
2. ⭐ **永远不要硬编码 API Key**
3. ⭐ **.env 文件是唯一来源**
4. ⭐ **环境变量会污染后续所有调用**
