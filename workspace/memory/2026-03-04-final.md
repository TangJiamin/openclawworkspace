## ⭐ 核心原则（必须记住）

### 1. Agent 核心文件职责分离（2026-03-04 17:13）

**每个文件有明确的职责**：

| 文件 | 谁读它？ | 目的 | 性质 |
|-----|---------|------|------|
| **SOUL.md** | Agent自己 | "我是谁" | 本质/个性 |
| **IDENTITY.md** | Agent自己 | "我叫什么" | 元数据 |
| **USER.md** | Agent自己 | "用户是谁" | 上下文 |
| **README.md** | 开发者/用户 | "如何使用我" | 技术文档 |
| **AGENTS.md** | Agent自己 | "同事是谁" | 协作文档 |
| **TOOLS.md** | Agent自己 | "用什么工具" | 参考手册 |

**关键区别**:
- **SOUL.md** = "我是谁"（本质、个性、行为准则）
- **IDENTITY.md** = "我叫什么"（名字、emoji、avatar）
- **USER.md** = "用户是谁"（偏好、项目、上下文）
- **README.md** = "如何使用我"（技术说明、配置）

**不应该**:
- ❌ 创建"中心化原则"文件（如 SYSTEM-PRINCIPLES.md、CORE-PRINCIPLES.md）
- ❌ 混淆文件职责
- ❌ 重复内容
- ❌ 破坏职责分离

**应该**:
- ✅ 每个文件有明确的职责
- ✅ Agent 自己读的文件 vs 开发者读的文件
- ✅ 单一数据源（SOUL.md 和 MEMORY.md）
- ✅ 不创建冗余的索引文件

## ⚠️ 错误和偏差

### 错误 1: 创建 SYSTEM-PRINCIPLES.md

**表现**: 创建了一个"中心化原则"文件来汇总所有原则

**问题**:
- 冗余 - 内容来自 SOUL.md 和 MEMORY.md
- 维护成本 - 需要同步更新
- 不是权威 - 真正的原则在源文件中

**改正**: 删除 SYSTEM-PRINCIPLES.md，直接引用 SOUL.md 和 MEMORY.md

### 错误 2: 创建 CORE-PRINCIPLES.md

**表现**: 又创建了一个类似的"核心原则"文件

**问题**: 和 SYSTEM-PRINCIPLES.md 一样的错误

**改正**: 删除 CORE-PRINCIPLES.md，更新 Agents 直接引用 SOUL.md 和 MEMORY.md

### 错误 3: 不理解文件职责分离

**表现**: 混淆了 SOUL.md 和 README.md 的职责

**问题**:
- SOUL.md 是 Agent 自己读的（灵魂）
- README.md 是开发者读的（技术文档）

**改正**:
- SOUL.md = Agent 的本质、个性、行为准则
- README.md = Agent 的技术说明、使用文档
