## Agent Output 统一管理工具

### 实施方案（v2.0 修正版）

**目录结构**（每个 Agent 目录下管理自己的产出）:
```
agents/
├── research-agent/

│   │   ├── task-20260312-092307/       # 时间戳格式: YYYYMMDD-HHMMSS
│   │   │   ├── .gitkeep

│   │   └── .archive/                   # 归档目录

├── content-agent/
│   ├── output/

│   └── ...
```

### 关键决策

1. **时间戳格式**: `YYYYMMDD-HHMMSS`（如 `20260312-092307`）
2. **元数据文件**: 不需要
3. **实施时间**: 立即开始

### 创建的工具

**agent-output-tool.sh** - 统一产出管理工具

**功能**:
- ✅ `create` - 创建产出目录（自动生成时间戳）
- ✅ `list` - 列出产出目录
- ✅ `clean` - 清理旧产出（默认保留7天）
- ✅ `archive` - 归档指定任务

**位置**: `/home/node/.openclaw/workspace/scripts/agent-output-tool.sh`

### 使用示例

```bash
# 创建产出目录
OUTPUT_DIR=$(bash /home/node/.openclaw/workspace/scripts/agent-output-tool.sh create research-agent)

# 列出所有 Agent 产出
bash /home/node/.openclaw/workspace/scripts/agent-output-tool.sh list

# 清理7天前的产出
bash /home/node/.openclaw/workspace/scripts/agent-output-tool.sh clean research-agent 7
```

### 优势

✅ **独立管理**: 每个 Agent 管理自己的产出，不依赖全局目录
✅ **归属清晰**: 产出一目了然是哪个 Agent 的
✅ **易于迁移**: 整个 Agent 目录可以直接移动
✅ **易于维护**: Agent 的所有内容（代码+产出）都在自己的目录下
✅ **符合直觉**: Agent 的产出自然应该在 Agent 的目录下
✅ **任务隔离**: 不同任务的产出完全分离
✅ **追溯性**: 任务ID包含创建时间

### 第一性原理分析

**表象**: 文件组织问题
**本质**: 产出的归属和管理边界
**最优解**: 产出的物理位置应该反映逻辑归属

**关键洞察**:
1. **归属清晰**: Agent 的产出就应该在 Agent 的目录下
2. **边界明确**: 每个 Agent 是独立的单元
3. **更易理解**: 新人一看就知道产出到哪里

## 🔧 visual-agent 配置优化（17:10）

### 问题发现

1. ✅ 配置有效 - `tools.subagents.tools` 配置正确
2. ✅ LLM 可以看到 exec 工具 - 测试 8 成功
3. ❌ 复杂任务失败 - Token = 0，超时

### 根本原因

**LLM 在复杂任务中进入"思考循环"**：
- 看到任务后开始"思考"如何完成
- 思考过程不计入 Token
- 思考时间过长，导致超时
- 最终没有输出（Token 0）

### 解决方案

**使用脚本而不是 LLM Agent**：
- ✅ Main Agent 直接调用脚本
- ✅ 不依赖 LLM 的理解能力
- ✅ 100% 可靠（已验证）

### 测试 9 的突破

**测试 9（17:06）的关键发现**：
- LLM 成功调用了 exec 工具
- 脚本正在运行（"Command still running"）
- 只是超时时间太短（2分钟不够）

### 配置更新

**已更新**：
