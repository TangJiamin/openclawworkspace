# Daily Summary - 每日记忆总结

## 目的

每天结束时生成一份总结，包括：
- 📈 **能力增长**：新学到的技能、工具、原则
- ⚠️ **不足之处**：错误、违背原则、理解偏差
- 💡 **改进建议**：如何避免重复错误、明天如何改进

## 工作流程

```
扫描今日记忆 (memory/YYYY-MM-DD.md)
  ↓
提取关键信息
  ↓
分析能力增长
  ↓
识别不足之处
  ↓
生成改进建议
  ↓
保存总结 (memory/daily-summary/YYYY-MM-DD.md)
  ↓
可选：发送重要提醒给用户
```

## 输入来源

- **今日记忆文件**：`memory/2026-03-04.md`
- **系统原则**：`SYSTEM-PRINCIPLES.md`
- **长期记忆**：`MEMORY.md`

## 分析维度

### 能力增长

**检测指标**：
- 新增的 Skill
- 学习的新工具（如 agent-reach）
- 理解更深入的原则
- 掌握的命令/操作

**提取方式**：
- 搜索 "新能力"、"学习到"、"掌握"
- 检查 Git commits（新增文件）
- 分析记忆内容

### 不足之处

**检测指标**：
- 违背原则的行为
- 用户的纠正反馈
- 重复的错误
- 理解偏差

**提取方式**：
- 搜索 "错误"、"违背"、"纠正"
- 检查用户反馈
- 分析失败的操作

### 改进建议

**生成方式**：
- 基于不足之处，生成具体建议
- 参考 SYSTEM-PRINCIPLES.md
- 结合长期记忆中的模式

## 自动化

### Cron 配置

**每日总结 + 自我反思（21:00）**:
```yaml
schedule: "0 21 * * *"
sessionTarget: "isolated"
payload: 扫描今日记忆，自我反思，生成学习计划
```

**定向学习（21:30）**:
```yaml
schedule: "30 21 * * *"
sessionTarget: "isolated"
payload: 基于总结，进行定向学习和实践
```

**深度回顾（23:59）**:
```yaml
schedule: "59 23 * * *"
sessionTarget: "isolated"
payload: 评估总结和学习完成情况，规划明天重点
```

**时间间隔**: 总结（21:00）→ 学习（21:30）间隔 30 分钟；学习（21:30）→ 回顾（23:59）间隔 2.5 小时

**核心原则**: 先总结，后学习 - 总结指导学习方向

**详见**: `memory/DAILY-LEARNING-SCHEDULE.md`

### 输出位置

- **总结文件**：`memory/daily-summary/YYYY-MM-DD.md`
- **周度汇总**：每周日汇总本周总结
- **月度归档**：每月归档到 `memory/archive/YYYY-MM/`

## 使用方式

### 手动触发

```bash
# 生成今天的总结
/path/to/skill --date today

# 生成指定日期的总结
/path/to/skill --date 2026-03-04
```

### 自动触发

- Cron 定时任务：每天 23:59
- 自动扫描今日记忆
- 自动生成总结
- 自动保存文件

## 质量标准

### 优秀总结的标准

1. **准确性**：正确识别能力增长和错误
2. **具体性**：每个点都有具体的例子
3. **可操作性**：改进建议可以实际执行
4. **诚实性**：不掩盖错误，不夸大成绩

### 常见错误

- ❌ 泛泛而谈，没有具体例子
- ❌ 只说好的，不提不足
- ❌ 改进建议不切实际
- ❌ 没有参考原则文档
