# storyboard-generator - 分镜表生成器

**描述**: 将剧本转换为分镜表（Shot List）和静帧提示词，支持即梦/banana模型

---

## 核心功能

将剧本转换为分镜表和静帧提示词，输出可直接用于图像生成的标准化数据。

---

## 核心原则

- **叙事层对齐**：每镜头只交代 1 个"唯一信息点"
- **镜头层对齐**：景别/机位/定格瞬间必须明确
- **画面层对齐**：构图、主体位置、光色、风格锚点一致
- **提示词三段式**：风格段 + 主体段 + 画面段，引用资产库

---

## 使用方法

```bash
bash /home/node/.openclaw/workspace/skills/storyboard-generator/scripts/generate.sh <script_file> <output_file>
```

### 参数说明

- `script_file`: 剧本文件路径（必需）
- `output_file`: 分镜表输出文件路径（必需）

---

## 输入要求

**必填**:
- 剧本正文（标准格式剧本）
- STYLE_BASE（风格锚点）
- 角色卡列表（带 @角色ID）
- 场景卡列表（带 SceneKey）

**可选**:
- 道具卡列表（带 PropKey）
- STYLE_VAR（风格变体：回忆/梦境等）
- 画幅（默认 16:9）
- 镜头密度（默认 18 镜头/集）

---

## 导演三帧分镜法（每场）

为每场生成 3 张关键帧：

### 起帧（KF_A）
- **目的**: 建立环境与关系
- **目的标签**: 信息/情绪/动作/反转/钩子（选1）
- **唯一信息点**: 一句话描述
- **定格瞬间**: 可截图的关键帧描述
- **景别/机位**: 全景/中景/近景/特写 + 平视/俯视/仰视/主观
- **主导运镜**: 静止/微推/拉远/轻摇/跟拍（选1）

### 爆帧（KF_B）
- **目的**: 冲突/动作爆点
- **格式**: 同起帧

### 钩帧（KF_C）
- **目的**: 悬念/转场/情绪余波
- **格式**: 同起帧

## 提示词三段式

```
最终提示词 = 风格段 + 主体段 + 画面段

┌──────────┐   ┌──────────┐   ┌──────────┐
│ 风格段    │ + │ 主体段    │ + │ 画面段    │
└────┬─────┘   └────┬─────┘   └────┬─────┘
     │              │              │
     ▼              ▼              ▼
┌──────────┐   ┌──────────┐   ┌──────────┐
│STYLE_BASE│   │@角色ID   │   │SceneKey  │
│风格一句话 │   │形象描述  │   │场景描述  │
└──────────┘   └──────────┘   └──────────┘
```

### 示例

**风格段**:
- 日系动漫风格，清新明亮，温馨治愈

**主体段**:
- @jl_chenfeng（少年，黑发黑瞳，身穿白色道袍），眼神坚定，手持长剑

**画面段**:
- scene_jl_yaoshougu（妖兽谷，奇花异草，雾气缭绕），中景，平视，主体居中偏左，背景模糊

**完整提示词**:
```
日系动漫风格，清新明亮，温馨治愈，@jl_chenfeng（少年，黑发黑瞳，身穿白色道袍），眼神坚定，手持长剑，scene_jl_yaoshougu（妖兽谷，奇花异草，雾气缭绕），中景，平视，主体居中偏左，背景模糊，高清细节
```

## 输出格式

### 1. 本集导演锚点总结

```
## 本集导演锚点

- **本集目标：**{一句话目标}
- **主情绪：**{紧张/热血/温馨/压抑}
- **钩子：**{前10秒抓手}
- **场景列表：**
  - 1-1 {场景名}
  - 1-2 {场景名}
- **主要角色：**@角色1, @角色2
```

### 2. 导演三帧分镜法（每场）

```
### 场次：{场号} {场景名}

#### 起帧 (KF_A)
- **镜头ID：** E{集}_S{场}_KF_A
- **目的标签：** 信息/情绪/动作/反转/钩子（选1）
- **唯一信息点：** {一句话}
- **定格瞬间：** {一句话，可截图}
- **景别/机位：** {全景/中景/近景/特写} + {平视/俯视/仰视/主观}
- **主导运镜：** 静止/微推/拉远/轻摇/跟拍（选1）

**提示词（三段式）：**
- 风格：{STYLE_BASE一句话}
- 主体：{@角色ID形象描述}，{本镜头表情动作}
- 画面：{SceneKey场景描述}，{景别}，{机位}，{构图}，{定格瞬间}
```

### 3. 镜头表（Shot List）

```
## 镜头表

| ShotID | SceneID | 景别 | 机位 | 运镜 | 目的 | 唯一信息点 | 定格瞬间 | 动作链 | 台词/OS | CharID | SceneKey | 优先级 | 状态 |
|--------|---------|------|------|------|------|------------|----------|--------|---------|--------|----------|--------|------|
| E01_S01 | 1-1 | 全景 | 平视 | 静止 | 信息 | {信息点} | {定格} | {动作} | {台词} | @xxx | scene_xxx | P0 | todo |
```

### 4. 静帧提示词清单

```
## 静帧提示词

### E01_S01

**三段式提示词：**
- 风格：{STYLE_BASE一句话}
- 主体：{角色形象}，{表情动作}
- 画面：{场景描述}，{景别}{机位}，{构图描述}，{主体位置}，{定格瞬间}

**完整提示词（直接复制用）：**
```
{STYLE_BASE}，{角色形象}，{表情动作}，{场景描述}，{景别}{机位}，{构图}，{定格瞬间}，高清细节
```

**负向提示词：**
文字、水印、logo、边框、二维码、现代建筑、现代服饰、枪械、车辆、血腥特写、内脏、喷溅、恐怖猎奇、低清晰度、模糊、噪点、过曝、透视错误、肢体畸形、多余手指、画面脏乱
```

### 5. QC 检查清单

```
## QC 检查

### P0（必须修复 - 影响剪辑）
- [ ] 是否缺建立/反应/转场镜头？
- [ ] 定格瞬间是否"一眼看懂"？
- [ ] 是否有轴线跳跃？
- [ ] 信息点是否清晰？

### P1（第二优先 - 影响一致性）
- [ ] 角色脸/服装是否漂移？
- [ ] 道具是否与角色卡一致？
- [ ] 场景锚点是否保持？

### P2（最后修复 - 美观问题）
- [ ] 画面是否足够干净？
- [ ] 光影是否统一？
- [ ] 是否足够电影化？
```
