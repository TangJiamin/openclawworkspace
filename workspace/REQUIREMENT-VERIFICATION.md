## 🎯 用户提出的功能需求

### 场景1: 按需生产

**需求描述**:
用户提出需求（如"生成小红书内容，推荐5个提升效率的AI工具"），系统自动完成：
1. 需求理解
2. 资料收集
3. 内容生产（文案）
4. 视觉生成（图片）
5. 视频生成（可选）
6. 质量审核
7. 输出结果

**关键要求**:
- ✅ 使用真实资讯（而非硬编码）
- ✅ 不同话题生成不同内容
- ✅ 所有 Agent 使用 sessions_spawn 协同

---

### 场景2: 定时批量生产

**需求描述**:
定时（如每天早上）自动执行：
1. 收集最新资讯
2. 识别热点话题
3. 批量生成内容（文案、图片、视频）
4. 批量审核
5. 筛选高质量内容
6. 生成待发布队列
7. 自动发布（或等待人工确认）

**关键要求**:
- ✅ 真实资讯驱动
- ✅ 批量生产多种内容类型
- ✅ 质量筛选机制

### 4. 数据传递

**验证**: ✅ **通过**

**方式**: 通过 task 参数传递

**示例**:
```bash
# research-agent 的输出
RESEARCH_OUTPUT="资讯内容..."

# 传递给 content-agent
CONTENT_OUTPUT=$(sessions_spawn \
  --agent-id "content-agent" \
  --task "基于以下资讯生成文案：
$RESEARCH_OUTPUT")
```
